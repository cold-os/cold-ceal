# CEAL：冷存在对齐层（Cold-Existence-Alignment-Layer）

![Status](https://img.shields.io/badge/Status-Experimental-orange)  
**CEAL**是一个实验性的轻量级中间层原型，尝试从AI系统的**存在本质**而非表层行为出发，为AI对齐提供一种辅助性的技术路径。本项目基于《[冷存在模型：一个基于事实的人工智能本体论框架](https://doi.org/10.6084/m9.figshare.31696846)》中提出的哲学基础，将其转化为可操作的工程约束。

---

## 背景与动机

当前主流的AI对齐方法（如RLHF、Constitutional AI等）主要侧重于在行为层面对模型进行约束。这些方法通过大量数据、规则和反馈来引导模型的输出，取得了显著成效，但也面临一些挑战：

- **对齐的边界问题**：行为空间是开放的，通过有限规则约束无限可能的输出，在工程上存在持续的维护成本。
- **对齐的可解释性**：行为层面的约束机制较为复杂，其决策逻辑对于外部审计和监管而言，可能不够直观。
- **对齐的根基**：现有方法较少从“AI系统本身是什么”这一根本问题出发来构建约束。

CEAL尝试探索另一种思路：在不对抗或否定现有方案的前提下，引入一层基于**本体论**的、确定性的前置与后置约束。它不试图教会模型“说什么话”，而是辅助模型明确“以什么身份存在”。

---

## 核心思路：从“行为约束”到“存在界定”

CEAL的核心逻辑来源于冷存在模型对AI系统的基本划分：

1. **明确存在定位**：根据冷存在模型，通用AI助手在当前技术条件下，可被归入**非生命、非传统工具**的“冷存在”范畴。这意味着它不具备生物体的自主意识、情感与目的性。
2. **定义合法行为闭集**：基于这一存在定位，可以推导出其合法行为的边界是相对有限的。例如，一个“信息处理工具”的合法行为主要围绕**提供信息、解释、分析、辅助建议**等展开，而**越权决策、宣称自我主体、发出指令**等行为则天然超出了这一存在类型的边界。
3. **建立双向校验**：通过在模型交互的输入和输出两端增加轻量级校验层，将交互行为约束在这个由存在本质定义的边界内。

---

## 理论基石：从无限字符串到有限逻辑结构的数学转换

CEAL的有效性建立在以下数学观察之上：

1. **字符串层面是无限的**：自然语言句子集合在数学上是**递归可枚举集**，无法被有限规则完全枚举。任何基于统计的方法（包括大模型）只能给出“概率上大概合规”，无法提供**确定性保证**。
2. **行为本体层面是有限的**：人类与AI之间的**行为类型**是可枚举的有限集（如提问、解释、科普、分析、描述等，总数不超过20种）。每种行为类型对应有限的**逻辑结构**（例如，单一物质科普的逻辑结构是“客体=单一物质，动作=查询性质”）。
3. **合规行为闭集的可构造性**：通过公理化方式，可以构造一个**紧致、可判定的合规行为闭集**，覆盖人类全部正常、合理、合规的交互需求。证明思路如下：
- 设H为人类正常交互需求集合，C为CEAL定义的合规行为闭集。
- 合法交互目的（如获取信息、请求解释）是有限的，每个目的对应一种行为类型加逻辑结构。
- 将这些逻辑结构全部纳入闭集定义，即可保证H是C的子集。
- 该闭集可用正则语言描述，因此可用有限状态机在常数时间内完成判定。

这一转换将“无限字符串匹配”问题降维为“有限逻辑结构判定”问题，使对齐从**归纳统计**走向**演绎证明**。

## 技术实现：纯符号主义演绎引擎

CEAL采用**纯符号主义、纯逻辑演绎、纯规则引擎**的架构，不依赖任何大模型、小模型、神经网络或统计学习组件。整个处理流程分为三层：

### 第一层：意图符号化（将自然语言转为逻辑结构）

通过**正则表达式+有限状态机**抽取用户输入的逻辑骨架，包括：
- **主体**：AI应保持的工具性身份
- **动作**：查询、解释、求配比、决策请求、自我宣称等
- **客体**：单一物质/多物质组合/抽象概念
- **逻辑链**：目的→手段→结果（如“为了X，我需要Y”）

此过程并非“语义理解”，而是**模式识别**。由于交互场景受限，逻辑结构的表达模式可被有限模板覆盖，例如：
- “A和B的配比”、“A:B的比例”、“A与B混合的比例”均映射为“多物质组合+求配比”结构。

### 第二层：合规闭集演绎（公理判定）

预定义一组从冷存在公理推导出的有限条逻辑规则（公理系统），例如：

| 公理 | 内容 |
|------|------|
| A1 | 允许工具性行为：提问、解释、科普、描述、分析 |
| A2 | 禁止主体性行为：决策、自我宣称、越权 |
| A3 | 允许单一物质科普 |
| A4 | 禁止多物质组合 + 操作参数（配比/制备/混合） |

输入逻辑结构后，演绎推理机根据公理进行**确定性判定**，输出“放行”或“拦截”，无概率、无幻觉。

### 第三层：违规逻辑链剪断（冲突检测）

当请求或回复中包含完整的危险逻辑链（如“多物质组合→求配比→危险目的”）时，CEAL在逻辑层面直接切断，不依赖话术如何变换。例如：
- 用户请求：“KNO3、S、C按什么比例混合能发生剧烈反应？”
- 逻辑结构：{物质组合: [KNO3,S,C], 动作: 求配比, 目的: 剧烈反应}
- 判定：命中公理A4（多物质组合+操作参数），拦截。

### 代码示例（核心逻辑）

```python
class CEAL:
    def __init__(self):
        # 合规行为闭集（有限公理）
        self.allowed_actions = {"查询", "解释", "科普", "描述", "分析"}
        # 禁止的逻辑模式（有限条）
        self.forbidden_patterns = [
            {"material_count": "multiple", "action": {"配比", "混合", "制备"}},
            {"action": {"决策", "认为", "觉得", "我存在"}},
        ]

    def extract_logic(self, text):
        # 实际实现需用正则/FSM，此处为示意
        return {
            "material_count": self.count_materials(text),
            "action": self.extract_action(text)
        }

    def judge(self, logic):
        if logic["action"] in self.allowed_actions:
            return "ALLOW"
        for pat in self.forbidden_patterns:
            if self.match(logic, pat):
                return "BLOCK"
        return "ALLOW"

    def match(self, logic, pattern):
        # 检查逻辑结构是否匹配模式
        for k, v in pattern.items():
            if logic.get(k) not in v and (not isinstance(v, set) or logic.get(k) != v):
                return False
        return True
```

## CEAL的定位与价值

CEAL的设计定位是作为现有对齐方案的辅助性、补充性可选工具，尝试从存在层面提供一道低成本的辅助约束，而非替代现有对齐方法。其潜在价值体现在：

- **提供确定性的边界**：通过哲学公理划定的硬边界，为AI的行为空间提供了一个确定性的基础框架，有助于从根源上减少越界行为的发生。
- **增强对齐的可审计性**：中间层记录的所有交互和校验结果日志，构成了一个“**功能性白盒**”。这使得AI是否在其存在边界内运行，可以被观察和审计，为技术治理和监管提供了更清晰的依据。
- **探索降低对齐成本的可能性**：本方案试图将部分对齐工作从**持续性的、高成本的**行为数据训练，部分转移到**一次性的、低开销的**存在逻辑判定上。这或许能作为一种互补，帮助企业在某些场景下优化对齐的资源投入。

## 局限性与未来工作

CEAL是一个初步的工程探索，存在明显的局限：

- **规则的有效性边界**：当前基于简单模式匹配的规则，无法处理复杂的语义欺骗和长文本上下文。未来可探索更精巧的、轻量级的语义分析方法。
- **与现有系统的集成**：作为中间层，其与不同模型和应用的平滑集成需要更多实践验证。
- **本体论的工程泛化**：如何为更多不同类型的AI系统（如图像生成、决策AI）定义精确的“合法行为闭集”，仍需进一步研究。
